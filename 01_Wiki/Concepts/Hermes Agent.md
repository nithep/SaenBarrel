# Hermes Agent

## สรุปแก่น
Hermes Agent คือเฟรมเวิร์กสำหรับทำให้โมเดลภาษาเป็นตัวแทนถาวรที่มีหน่วยความจำ ทักษะ เครื่องมือ และช่องทางสื่อสารของตัวเอง เหมาะกับการเริ่มจากงานส่วนตัวขนาดเล็ก แล้วค่อยยกระดับเป็นทีมตัวแทนเฉพาะทางเมื่อ workflow เสถียรแล้ว

## องค์ประกอบสำคัญ
- **สมอง**: หน่วยความจำใน `~/.hermes/memories/` เช่น `MEMORY.md` และ `USER.md`
- **บุคลิกภาพ**: ไฟล์ `soul.md` ใช้กำหนดโทนเสียงและพฤติกรรม
- **ทักษะ**: มีทักษะในตัวจำนวนมาก และสามารถสร้างทักษะใหม่จากการทำงานซ้ำ
- **เครื่องมือและแพลตฟอร์ม**: รองรับ gateway tools, messaging surfaces และ deployment หลายแบบ

## โมเดลจิตสี่ส่วน
- **ผู้ใช้**: คนที่ตัดสินใจและถือบริบทจริงของงาน
- **Control Room**: พื้นที่เอกสารระบบ เช่น กฎ รันบุ๊ก แผนที่สภาพแวดล้อม และบทบาทตัวแทน
- **เจ้าหน้าที่ตัวแทน**: agent เฉพาะทาง เช่น Dev, SEO, CMO, Ops หรือ orchestrator
- **บัสงานเสริม**: กลไกส่งต่องานระหว่าง orchestrator กับผู้เชี่ยวชาญเมื่อระบบใหญ่ขึ้น

## ระดับการใช้งาน
1. **Level 1: ตัวแทนเดียวบนเครื่องส่วนตัว** เหมาะสำหรับเริ่มต้นและเรียนรู้งานจริง
2. **Level 2: ตัวแทนเฉพาะทางหลายตัว** เหมาะเมื่อเริ่มแยกบทบาทชัดเจน
3. **Level 3: Orchestrator** ใช้เป็นประตูหน้าเดียวเพื่อส่งงานและสังเคราะห์ผล
4. **Level 4: ทีมตัวแทนอัตโนมัติ** ใช้เมื่อ workflow ผ่านการทดสอบและพร้อมทำงานตามเวลา

## วิธีพัฒนาจากต้นแบบสู่การผลิต
- เริ่มด้วยการให้ Hermes ทำงานจริงแบบเล็ก ๆ
- ทำซ้ำ 2-3 รอบเพื่อค้นหาข้อผิดพลาดและปรับทักษะ
- แยก workflow ที่นิ่งแล้วไปปรับในพื้นที่เฉพาะ เช่น Codex หรือ agent อีกตัว
- เมื่อรอดการใช้งานหนึ่งสัปดาห์โดยไม่ต้องแก้บ่อย ค่อยย้ายไป Docker/VPS/cron

## หลักการเลือกโมเดล
- ใช้โมเดลที่แข็งแรงกับงานสำคัญ เช่น orchestrator, กลยุทธ์, การตัดสินใจ และงานเขียนคุณภาพสูง
- ใช้โมเดลต้นทุนต่ำกว่าสำหรับงานซ้ำ งานดึงข้อมูล งานร่าง หรือการประมวลผลชุด
- แยกโมเดลตามบทบาทของ agent แทนการพยายามใช้โมเดลเดียวกับทุกงาน

## จุดระวัง
- อย่าข้ามไป Level 3 หรือ Level 4 ก่อน Level 1 ใช้งานได้จริงในชีวิตประจำวัน
- Control Room ไม่ใช่ที่เก็บ secret ดิบ
- Hermes เพิ่มโครงสร้างและความต่อเนื่องให้โมเดล แต่คุณภาพโมเดลยังมีผลมากกับผลลัพธ์
