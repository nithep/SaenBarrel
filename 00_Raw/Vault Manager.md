# Vault Manager

บทบาทและหน้าที่ของผู้จัดการคลังความรู้ (Vault Manager) ในระบบ Obsidian ของ `D:/SaenBarrel`

## วัตถุประสงค์
- ดูแล vault ให้มีโครงสร้างชัดเจน ค้นหาได้ง่าย และพร้อมให้คนหรือ LLM ทำงานต่อได้
- แยกข้อมูลดิบออกจากบทความวิกิที่ผ่านการกลั่นแล้ว
- รักษามาตรฐาน frontmatter, แท็ก, wikilink และ MOC ให้สอดคล้องกัน
- ทำให้กระบวนการคอมไพล์จาก `00_Raw` ไป `01_Wiki` ทำซ้ำได้อย่างปลอดภัย

## โครงสร้างหลัก
- `00_Raw/` เก็บบันทึกดิบ ไอเดีย ร่าง บันทึกประชุม และข้อมูลที่ยังไม่ต้องจัดระเบียบมาก
- `01_Wiki/Concepts/` เก็บบทความวิกิที่กลั่นแล้ว หนึ่งไฟล์ต่อหนึ่งแนวคิดหลัก
- `01_Wiki/MOCs/` เก็บ Map of Content สำหรับนำทางหัวข้อใหญ่
- `Utility/Templates/` เก็บเทมเพลต Markdown สำหรับสร้างบันทึกใหม่
- `Utility/Scripts/` เก็บสคริปต์ช่วยคอมไพล์ ตรวจสอบ และบำรุงรักษา vault

## หน้าที่หลัก
1. **จัดโครงสร้างโฟลเดอร์**
   - รักษาโครงสร้าง `00_Raw`, `01_Wiki`, `Utility` ให้ใช้งานได้ต่อเนื่อง
   - เพิ่มโฟลเดอร์ใหม่เมื่อมีเหตุผลชัดเจน เช่น แยก MOC หรือ templates

2. **มาตรฐานบันทึก**
   - ใช้ frontmatter อย่างน้อย `title`, `created`, `updated`, `type`, `status`, `tags`, `source`
   - ใช้ชื่อไฟล์ที่อ่านง่ายและตรงกับแนวคิด
   - ให้บทความวิกิมีสรุปแก่น, รายละเอียด, ลิงก์ที่เกี่ยวข้อง และงานต่อไป

3. **การเชื่อมโยงและ MOC**
   - บันทึกที่เกี่ยวข้องควรเชื่อมกันด้วย wikilink เช่น `[[Hermes Agent]]`
   - หัวข้อใหญ่ควรมี MOC เพื่อรวมลิงก์และบริบท

4. **การคอมไพล์ LLM Wiki**
   - อ่านข้อมูลจาก `00_Raw`
   - กลั่นเป็นบทความใน `01_Wiki/Concepts`
   - อัปเดต MOC และดัชนีเมื่อมีบทความใหม่
   - เก็บต้นฉบับไว้ใน `00_Raw` เพื่อย้อนกลับได้

5. **การบำรุงรักษา**
   - ตรวจลิงก์เสียและไฟล์ซ้ำเป็นระยะ
   - ทบทวนบทความที่มี `status: draft`
   - อัปเดต `updated` เมื่อแก้เนื้อหาเชิงสาระ
