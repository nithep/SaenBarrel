# แนวทางปฏิบัติ LLM Wiki

## วัตถุประสงค์
ใช้แนวทางนี้เพื่อสร้างและจัดการองค์ความรู้จากโมเดลภาษาขนาดใหญ่ (LLM) ในรูปแบบวิกิที่เชื่อมโยงกันได้ง่ายภายใน Obsidian vault

## โครงสร้างโฟลเดอร์หลัก
- `00_Raw` : สำหรับเก็บข้อมูลดิบ ข้อความสนทนา ผลลัพธ์จาก LLM ที่ยังไม่ผ่านการประมวลผล
- `01_Wiki` : สำหรับเก็บบทความวิกิที่ผ่านการคัดเลือก สรุป เชื่อมโยง และจัดรูปแบบแล้ว

## การทำงานทั่วไป
1. **รวบรวมข้อมูลดิบ**  
   - บันทึกผลลัพธ์จากการสนทกับ LLM หรือการค้นหาข้อมูลลงใน `00_Raw` เป็นไฟล์ markdown โดยใช้ชื่อที่อธิบายได้ชัดเจน เช่น `YYYY-MM-DD หัวข้อ.md`
   - รักษารูปแบบ frontmatter พื้นฐานเพื่อการค้นหาและกรองในอนาคต

2. **ประมวลผลและสรุป**  
   - อ่านไฟล์จาก `00_Raw` สรุปสาระสำคัญ แยกประเด็น และเขียนใหม่ในรูปแบบบทความวิกิ
   - บทความวิกิควรมี:
     - หัวเรื่องชัดเจน (เหมือนชื่อบทความในวิกิพีเดีย)
     - ส่วนบทนำสั้น ๆ
     - หัวข้อย่อยตามความจำเป็น
     - ลิงก์ไปยังบทความที่เกี่ยวข้องภายใน `01_Wiki` ด้วยรูปแบบ `[[ชื่อบทความ]]`
     - แท็ก (#) สำหรับหมวดหมู่หรือธีมข้ามบทความ

3. **จัดเก็บบทความวิกิ**  
   - บันทึกบทความที่เสร็จสมบูรณ์ลงใน `01_Wiki` โดยใช้ชื่อบทความเป็นชื่อไฟล์ (แทนที่ช่องว่างด้วย _ หรือเว้นวรรคตามความชอบ แต่ให้สอดคล้องกับการใช้วิกิลิงก์)
   - ตรวจสอบให้แน่ใจว่าไม่มีไฟล์ซ้ำชื่อ หากมีให้พิจารณาการรวมหรือลงวันที่ลงท้าย

4. **เชื่อมโยงและสร้างแผนที่เนื้อหา (MOC)**  
   - สร้างบทความพิเศษประเภท Map of Content (ตัวอย่างเช่น `[[แผนที่เนื้อหา LLM]]`) เพื่อแสดงภาพรวมและเชื่อมโยงไปยังบทความย่อย
   - ปรับปรุง MOC เป็นระยะเมื่อมีบทความใหม่เพิ่ม

5. **การบำรุงรักษา**  
   - ตรวจสอบลิงก์ที่เสียเป็นประจำ
   - รวมบทความที่ซ้ำซ้อนหรือแยกบทความที่เนื้อหามากเกินไป
   - อัปเดต frontmatter และแท็กเมื่อมีการจำแนกประเภทใหม่

## ตัวอย่าง Frontmatter (ยืดหยุ่น)
```yaml
---
title: "ชื่อบทความ"
created: 2026-06-05
tags: [llm, ปัญญาประดิษฐ์, วิกิ]
related:
  - [[บทความที่เกี่ยวข้อง 1]]
  - [[บทความที่เกี่ยวข้อง 2]]
---
```

## เคล็ดลับการใช้งานกับ LLM
- ใช้คำสั่งเฉพาะเพื่อขอให้ LLM สร้างสรุปในรูปแบบ markdown ที่สามารถคัดลอกวางได้ทันที
- ขอให้ LLM เสนอชื่อบทความและแท็กที่เหมาะสมตามเนื้อหา
- ตรวจสอบและแก้ไขข้อผิดพลาดหรือข้อมูลที่ล้าสมัยก่อนบันทึกเป็นวิกิ

## การเชื่อมต่อกับ Hermes Agent
- สามารถเรียกใช้ `llm-wiki` skill เพื่อค้นหา สร้าง หรืออัปเดตบทความใน `01_Wiki` ได้โดยตรงจากแชท
- ตรวจสอบให้แน่ใจว่าตัวแปรแวดล้อม `OBSIDIAN_VAULT_PATH` ชี้ไปที่ `D:/SaenBarrel` เพื่อให้ skill ทำงานได้ถูกต้อง
