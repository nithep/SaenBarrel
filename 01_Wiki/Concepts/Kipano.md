# Kipano — ระบบจดบันทึกของ CEO Obsidian

> Stefango (Kipano) — CEO ของ Obsidian MD — ใช้ระบบจดบันทึกที่ "ขี้เกียจและวุ่นวาย" แต่ทรงพลัง เน้นเครือข่ายความรู้ผ่านความยืดหยุ่นมากกว่าการจัดระเบียบเข้มงวด

## หลักการพื้นฐาน

### 1. File over App
บันทึกต้องอยู่ในรูปแบบไฟล์ Markdown ที่เป็นอิสระจากซอฟต์แวร์ — ข้อมูลคงอยู่แม้ซอฟต์แวร์จะเลิกพัฒนา

### 2. การปฏิเสธโฟลเดอร์
หลีกเลี่ยงการใช้โฟลเดอร์จัดหมวดหมู่ เพราะบันทึกหนึ่งเรื่องมักเกี่ยวข้องกับหลายหัวข้อ ใช้ **Properties** และ **Links** แทน

### 3. ความไร้ระเบียบที่ก่อให้เกิดเครือข่าย
การปล่อยให้บันทึกอยู่ในที่เดียวกันช่วยให้เห็นการเชื่อมโยงที่คาดไม่ถึง

## โครงสร้างไฟล์

| โฟลเดอร์ | วัตถุประสงค์ |
|----------|------------|
| Root | บันทึกที่เขียนเอง, Journal, Essays, Evergreen Notes |
| References | ข้อมูลจากภายนอก (People, Books, Clippings) |
| Utility | Attachments, Templates, Daily notes |

## เครื่องมือสำคัญ

| เครื่องมือ | หน้าที่ |
|-----------|---------|
| **Templates** | ใส่ Metadata อย่างรวดเร็ว |
| **Unique Notes** | บันทึกความคิดทันทีด้วยชื่อเป็นวันเวลา |
| **Internal Links** | "กฎของ Steph" — เชื่อมโยงทุกคำที่กล่าวถึงเป็นครั้งแรก |
| **Backlinks** | ดูว่าบันทึกปัจจุบันถูกอ้างอิงจากที่ใด |
| **Evergreen Notes** | สรุปแนวคิดเป็นบันทึกชิ้นเดียวที่เชื่อมโยงซ้ำได้ |

## Rating System (1-7)
- **7** = มีอิทธิพลต่อชีวิตอย่างมหาศาล
- **1** = มีอิทธิพลในทางลบ

## วงจรการทำงาน

| ระยะเวลา | งาน |
|----------|------|
| รายวัน | จดบันทึกผ่าน Unique Notes, เชื่อมโยงถึงบุคคล/สถานที่/แนวคิด |
| ทุก 3-4 วัน | ตรวจบันทึก → รวบรวม → สรุปประเด็น |
| รายสัปดาห์ | วางแผนงานแบบเรียบง่าย |
| รายเดือน | สะท้อนคิด (Reflection) + อัปเดต MOC |
| ทุก 2-3 เดือน | สุ่มเปิดบันทึกเก่า (Open Random Note) |
| รายปี | ทบทวนสรุปรายเดือนผ่านคำถาม 40 ข้อ |

## ข้อสรุป

> ความสำเร็จไม่ได้อยู่ที่ความเป็นระเบียบของไฟล์ แต่อยู่ที่การสร้าง "เครือข่ายความรู้" ที่ช่วยให้เห็น "ร่องรอยของการคิด" (Tracks in Thinking)
